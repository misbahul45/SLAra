**`road_geometry` per kandidat M4 (per 16 Jul 2026).** Tiap objek di `candidates[]` kini membawa
field opsional **`road_geometry`** — array `[lat, lng]` yang **mengikuti jaringan jalan** (hasil
*snap* OSRM `route`, `overview=simplified`). Field `geometry` lama **tidak berubah** (tetap titik stop
tur, dipakai untuk marker & urutan). Geometri di-*precompute* di build-time dan disimpan ke
`services/ai/data/pareto_routes_*.json` — **tidak** ada panggilan OSRM saat runtime (konsisten ADR-004);
endpoint M4 tetap menyajikan JSON apa adanya. Regenerasi: `python services/ai/scripts/snap_routes_to_roads.py`
(default OSRM publik; `--server http://localhost:5000` untuk OSRM self-host). FE **fail-soft**: kalau
`road_geometry` absen, garis jatuh balik ke `geometry` (lurus antar-stop).

## C. Error format (semua endpoint)

```json
{ "error": { "code": "MODEL_UNAVAILABLE", "message": "M4 optimizer timed out, served cached Pareto set", "degraded": true } }
```

HTTP 200 + `degraded: true` untuk graceful degradation (sesuai failure cascade design); 4xx/5xx hanya untuk kegagalan nyata.

> **Catatan implementasi §C:** degradasi M2 saat ini **tidak** memakai amplop `error` di atas —
> M2 mengembalikan HTTP 200 dengan flag `m2_degraded: true` di body utama (lihat B-bis). Semangatnya
> sama (degradasi = 200, bukan 5xx), bentuknya beda. `GET /internal/m4/routes` dengan scenario tak
> dikenal mengembalikan **404** (kegagalan nyata, bukan degradasi) — konsisten dengan §C.
